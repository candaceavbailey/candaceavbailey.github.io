Rockbuster Stealth Film Rental Analysis

Answering business questions with SQL queries to help Rockbuster Stealth's business intelligence (BI) department with the launch strategy for their new online video service.

Objective

Load all of Rockbuster's data into SQL. Write moderately complex queries to analyze data and provide data-driven answers to the following questions proposed by the Rockbuster Stealth Management Board.

  • Which movies contributed the most/least to revenue gain?
  • What was the average rental duration for all videos?
  • Which countries are Rockbuster customers based in?
  • Where are customers with a high lifetime value based?
  • Do sales figures vary between geographic regions?

Understanding Rockbuster's Data Structure

Before analysis can begin, it's important to interpret how Rockbuster's tables are related to one another. Writing out table storage structures, including keys and indexes helps to connect Rockbuster's data. The addition of a data schema turns the written table descriptions into a connected visually showing where there are connections between tables.

Rockbuster's databases are structured in a snowflake schema, meaning their fact tables have multiple dimensions that branch out to create greater detail.

Joining Tables

With a snowflake schema multiple fact tables need to be connected, in SQL, using primary and foreign keys before results can be filtered. Each question posed by Rockbuster stakeholders involved joining two or more tables to answer.Taking time to understand the connections between tables and creating a data dictionary was an import precursor to analysis.

The first step to answering the question which countries customers are based in was to find how many unique values are in the Country table, which produced a result of 109 countries. With this information alone decision cannot be made in regard to how to distribute marketing budgets for different films.

The Country and Customer table can be joined using the primary and foreign keys present in the Address and City tables. The COUNT function creates a calculated total customer customer for each grouped country listed.
Grouping the query results by country makes the resulting table easier to understand. The final result showed the top countries in Rockbuster's database in terms of customer count were India, China, United States, Japan, Mexico, Russia, Brazil, Philippines, Turkey, and Indonesia.
